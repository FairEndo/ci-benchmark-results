About the Benchmarks

This dashboard compares CI runner performance using standardized benchmarks. Runners are grouped into hardware tiers by OS so you're always comparing like with like. Each benchmark run executes the same workload multiple times and reports the median score for stability.

⚡ CPU Score

Measured with sysbench cpu. Counts how many prime-number computation events the runner completes per second. Higher is better—it reflects raw single-threaded compute throughput.

💾 Memory Throughput

Measured with sysbench memory. Reads and writes sequential blocks of memory as fast as possible, reported in MiB/sec. Higher is better—it reflects memory bandwidth available to your CI jobs.

⏱ Timing & Queue

Tracks the full lifecycle of each CI job—from trigger to completion. Queue time is how long the job waited before a runner picked it up. Run time is actual execution. Both matter for real-world CI speed.

📈 Trend Tracking

Every run is recorded over time so you can spot regressions or improvements. The timeline chart plots CPU median scores across all runs, grouped by provider and runner.

Performance Over Time

CPU median scores (events/sec) plotted over time for every provider/runner combination. Use this to spot performance regressions, improvements, or variability across runs. Each point is one benchmark run; lines connect consecutive runs for the same runner. Hover for details.
